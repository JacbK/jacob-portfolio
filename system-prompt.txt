# Site-in-a-Box: System Architecture

## Project Overview

This is an AI-powered personal portfolio template. The site is designed to be "self-assembling" - an AI agent researches the user and populates the website automatically.

## Architecture

```
site-in-a-box/
├── .agent/                    # AI Agent Configuration
│   └── instructions.md        # Agent brain - research & population instructions
│
├── bin/                       # CLI Tools
│   └── setup.sh              # Bootstrap script to initialize the agent
│
├── src/
│   ├── app/                  # Next.js App Router
│   │   ├── layout.tsx        # Root layout with fonts/metadata
│   │   ├── page.tsx          # Main page - conditional rendering
│   │   └── globals.css       # Tailwind imports + custom styles
│   │
│   ├── components/
│   │   └── ui/               # Skeleton UI Components
│   │       ├── Hero.tsx      # Hero section with name, title, tagline
│   │       ├── BentoGrid.tsx # Project showcase grid
│   │       └── Terminal.tsx  # Stylized terminal for bio/about
│   │
│   └── data/
│       ├── schema.ts         # TypeScript interfaces for user data
│       └── user.json         # Populated by AI agent (initially empty)
│
├── mcp-requirements.json      # MCP server configuration
├── system-prompt.txt          # This file - architecture overview
└── README.md                  # User-facing documentation
```

## Data Flow

1. **User runs `bin/setup.sh`**
   - Installs dependencies
   - Launches Claude Code with `.agent/instructions.md`

2. **Agent researches user**
   - Uses MCP tools (if configured) to search web
   - Gathers information from GitHub, LinkedIn, etc.
   - Asks user clarifying questions

3. **Agent populates `src/data/user.json`**
   - Follows schema defined in `src/data/schema.ts`
   - Validates JSON structure

4. **UI renders populated data**
   - Components read from `user.json`
   - Conditional rendering handles empty states

## Component Architecture

### Hero.tsx
- Displays: name, title, tagline, location
- Social links from contact data
- Call-to-action buttons
- Animated entrance with Framer Motion

### BentoGrid.tsx
- Displays: projects array
- Flexible grid layout (2-4 columns)
- Project cards with tech stack pills
- Featured projects highlighted

### Terminal.tsx
- Displays: bio.long
- Typewriter effect for text
- Terminal styling (green text, dark bg)
- Command prompt aesthetic

## Styling System

- **Tailwind CSS v4**: Utility-first styling
- **Framer Motion**: Animations and transitions
- **clsx**: Conditional class composition
- **Lucide React**: Icon library

## MCP Integration

The agent can use these MCP servers for research:

- **Google Search**: General web queries
- **Exa**: Deep semantic search
- **GitHub**: Repository and contribution data
- **Firecrawl**: Web scraping for detailed data

Configure in `mcp-requirements.json` or Claude Code settings.

## Key Files for LLMs

When reading this codebase, prioritize:

1. `src/data/schema.ts` - Understand the data structure
2. `.agent/instructions.md` - Agent behavior specification
3. `src/app/page.tsx` - Main page composition
4. `src/components/ui/*.tsx` - UI component implementations

## Development Commands

```bash
npm run dev      # Start development server (localhost:3000)
npm run build    # Production build
npm run start    # Start production server
npm run lint     # Run ESLint
```

## Empty State Handling

When `user.json` is empty or missing:
- Hero shows "Setting up your portfolio..."
- BentoGrid shows skeleton placeholders
- Terminal shows "Initializing..." animation

## Extending the Template

To add new sections:

1. Define types in `src/data/schema.ts`
2. Create component in `src/components/ui/`
3. Add conditional rendering in `src/app/page.tsx`
4. Update `.agent/instructions.md` with data requirements
